3.工作流程

发表于2024-12-23|更新于2024-12-27|webpack4
|阅读量:

手动打包

我们一般都是执行npm run build进行打包, 实际上webpack本质就是一个Compiler类,导出的是一个实例.我们可以手动调用实例进行打包.Node 接口文档

stats.toJson

  • 返回是一个对象, 包含打包的文件信息、内容、依赖模块等信息

现在我们来手动实现下webpack实例来了解webpack的工作流程

工作流程

初始化参数

  • 从配置文件和shell语句中读取与合并参数，得出最终的参数

初始化实例

  • 用上一步得到的参数配置，初始化生成一个Compiler实例

循环注册插件

  • 读取配置中的plugins选项，循环并注册插件 (此时插件只是注册(添加到待执行队列中),没有执行),往插件传入compiler实例
  • 插件调用实例的订阅方法, 将回调函数加入实例队列中等待后续执行, 实际webpack依赖的是tapable库,这里咱们自己实现了tapable库我们可以看到上面就是一个发布订阅模式

执行实例compiler的run方法，开始编译

我们来分步骤讲解下

从配置读取入口路径

获取入口内容调用loader编译并存入modules数组中

  • loader本质上是一个函数, 就是转换内容,如babel-loader转换es6代码为es5代码
  • 转换后定义模块对象, 模块对象包含id路径、source源代码、name入口来源信息, 存入modules数组中

将编译代码转ast获取依赖模块

  • 通过acorn库将loader转换后的代码转换为ast,然后识别ast, 收集依赖关系,从而形成一颗关系树.比如index.js依赖了require(‘a.css’)
  • cssLoader是定义的loader, 模拟css-loader处理css文件, 得到js对象, 存入modules数组中

将模块转换为chunk

5. 根据chunk构建输出js对象

  • 将chunk转换为一个个单独的文件加入到输出列表对象(这步是修改输出内容的最后机会),该对象就是输出文件名、内容

执行插件

  • 执行发布订阅模式的发布, 将输出文件对象传入进去供插件操作修改

遍历输出对象，写入文件系统到打包目录

总结

  • 初始化参数: 从配置文件和 Shell 语句中读取与合并参数，得出最终的参数。
  • 初始化实例: 定义config、 初始化发布订阅类供插件注册用
  • 从配置读取插件循环订阅注册
  • 执行compile.run开始编译
    • 获取入口文件路径同时fs读取入口内容进行loader转换
    • 将loader转换后的代码通过acorn转ast获取依赖模块, 如果依赖模块不是js也要调用loader进行转换
    • 构建输出对象: 即chunk对象(即要打包输出的js), chunk对象包含一个入口的所有module模块数组(入口模块和依赖模块)、打包输出的文件名id. 注意: 一个入口一个chunk, 一个chunk包含多个依赖module模块. 这一步是修改输出内容的最后机会,我们可以通过插件来修改
    • 执行插件: 传入构建对象,此时可以修改输出内容
    • 遍历输出对象,操作fs写入文件系统(打包文件夹)
