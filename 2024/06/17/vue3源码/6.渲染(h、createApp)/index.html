6.渲染(h、createApp)

发表于2024-06-17|更新于2024-06-18|vue3源码

基本使用

运行结果如下图:

  • h函数: 用来创建虚拟节点
  • setup如果返回一个对象, 则会作为data属性使用;如果返回一个函数, 则会当成render函数使用,不会触发外层的render函数
  • render函数中的this以及参数proxy是一样的,都是组件实例对象(上下文)
  • createApp接收两个参数: 1.组件对象 2.组件的属性props, 返回一个对象,对象有一个方法mount,用来挂载组件

下面我们通过渲染流程来讲解createApp和h函数

createApp

createApp(App, {test: 1,}).mount("#app")
createApp主要做了以下几件事情:

  1. 定义渲染函数-render函数
    • 里面就是定义了render方法即render里面执行了patch方法 patch(null, vnode, container) // 1.旧的虚拟节点 2.新的虚拟节点 3.容器,该方法主要是将虚拟节点渲染到真实dom上(通过一些js操作dom的方法)
  2. 定义了mount方法
    • 获取dom节点: container = nodeOps.querySelector(container)
    • 清空容器: container.innerHTML = ''
    • 挂载
      • 创建vnode: let vnode=createVNode(rootComponent, rootProps)
      • 执行上面的render方法进行渲染: render(vnode, container)

创建组件vnode

执行 let vnode = createVNode(rootComponent, rootProps)

  • createVNode函数中判断type是对象,说明是组件,构造了一个vnode对象

    执行createApp定义的render函数挂载组件

执行 render(vnode, container)

判断节点类型是组件,进行组件挂载

创建组件实例

将vnode的属性值赋值到组件实例对象上

创建effect让render函数执行

执行render方法

将render返回的vnode渲染到页面上

修改标识位isMounted

instance.isMounted = true; // 组件挂载完成

总结

流程比较长,需要对照着代码理解

  1. createApp()方法定义渲染函数-render函数
  2. 调用$mount()方法执行
    • 获取dom节点: container = nodeOps.querySelector(container)
    • 清空容器: container.innerHTML = ''
    • 挂载
      • 创建组件vnode: let vnode=createVNode(rootComponent, rootProps)
      • 执行上面的render方法进行渲染: render(vnode, container),render内部即patch(null, vnode, container) // 1.旧的虚拟节点 2.新的虚拟节点 3.容器
  3. 创建组件vnode
    • 定义vnode结构,以及type标识shapeFlag是组件/元素字符串等
    • 执行上面的render方法->patch函数->通过shapeFlag判断是组件执行组件挂载mountComponent
  4. 组件挂载mountComponent
    • 创建组件实例. createComponentInstance(上面的vnode)
    • 解析数据到组件实例对象: 将vnode的属性值赋值到组件实例对象上.
      • 1.先将props、data、children赋值到实例上
      • 2.在instance上绑定proxy用于代理ctx上下文,当获取属性时候直接返回,如proxy.state.name即返回instance.ctx._.setupState[key] . instance.proxy = new Proxy(instance.ctx, componentPublicInstance as any)
      • 3.判断setup返回值,如果是函数则赋值到instance.render,如果是对象就是data,否则instance绑定外层的render, 没有没有render且有template,编译template,也就是说render优先级>template
    • 创建一个effect让instance.render函数执行
  5. 创建一个effect让instance.render函数执行,使其响应式.此时的effect为渲染effect
    • 判断instance.isMounted: 是否挂载完成. 第一次肯定为false,否则为更新
    • 执行instance.render,并传入instance.proxy且this也指向instance.proxy
      • render里面就是h函数,相当于调用createVNode创建vdom
      • 对render返回的vdom进行patch操作,将vdom渲染到页面上(判断是元素、文本、组件等)
    • 还原标志位 instance.isMounted = true; // 组件挂载完成
  6. effect出栈

h函数

h函数就是createVNode函数, h函数内部就是调用了createVNode函数 h函数

所以:

  • 第一个参数type: 组件或者原生标签
  • 第二个参数props: 属性、children、默认插槽
  • 第三个参数children: 子节点,可以为数组、字符串、具名插槽,数组中可以是h函数、字符串

createApp函数只能传前2个参数,type是组件,第二个参数是组件的属性
